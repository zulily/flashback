flashback¶

Overview¶

flashback, is a simple command-line utility that offers fast system and configuration file recovery on remote hosts.

It is a tool that is generally used for archiving smallish files just prior to the roll-out of configuration changes, and only one version of the file or files may be archived per day. If subsequent archive commands are run in the same day, only the latest version persists.

Example Usage¶

Archive /etc/rsyslog.conf¶

Let’s say we are about to roll out significant changes to /etc/rsyslog.conf through some other mechanism (e.g. chef or SaltStack), and we are very concerned about the prospect of something going wrong and losing several minutes of log data. Let’s perform a backup which will allow us to recover near instantly, in the event something goes wrong, such as a syntax error that was not caught.

$ flashback archive -H localhost -F /etc/rsyslog.conf

Note

-H may be specified multiple times, but more commonly, consider using -f hosts.txt, where hosts.txt is a file containing a list of hosts to act upon, one hostname per line.

Diff Two Versions¶

Diff the current and archived version (from earlier in the same day, assuming it exists) of /etc/rsyslog.conf

$ flashback diff -F /etc/rsyslog.conf -H localhost

Note

Any two versions (there is a max of one archive per file, per day) may be compared. Please use the –date-first and –date-second arguments.

Recover¶

Continuing with the /etc/rsylog.conf example, let’s say something did go wrong, and we need to recover the latest version (archived today) immediately. We can recover the file and perform an rsyslog restart.

$ flashback recover -H localhost -F /etc/rsyslog.conf \
  --post-recover-command="sudo service rsyslog restart"

Report¶

It may be necessary to get a precise idea of exactly what has been archived on a server, or across several servers. The report subcommand provides a hierarchical listing, broken down by date.

$ flashback report -H localhost -f /etc/rsyslog.conf

Purge¶

It may be desirable to delete all archived files, which may be accomplished by using the purge subcommand.

$ flashback report -H localhost

Important Considerations¶

  • Only one archive of any file may exist per day, re-running the archive subcommand will overwrite any previous archives from earlier in the day
  • Backed up file names must be unique, or a collision may occur. For example, archiving both /etc/rsylog.conf and /usr/local/etc/rsyslog.conf will result in the last file to be archived overwriting the first one.
  • If not using the default archive directory (/root/.flashback), precautions should be taken to ensure sensitive file archives are properly protected.

Command-line Reference¶

Roll-back changes in a flash. Perform archiving, recovery and reporting for small system and configuration files. Can be a bit slow to execute if used in a way that opens up large numbers of concurrent ssh connections

usage: flashback [-h] {archive,diff,purge,recover,report} ...
Sub-commands:
archive

Archive system files

usage: flashback archive [-h] [--verbose]
                         (--host hostname | --hosts-file FILE)
                         [--system-file FULLPATH] [--archive-directory N]
                         [--parallel-workers N] [--sudo-password-prompt]
Options:
--verbose=False, -v=False
 Turn on verbose fabric output.
--host, -H Hostname for host where we wish to archive files or perform reporting
--hosts-file, -f
 Text file containing a list of hosts, one per line. Ignored if –host is specified
--system-file, -F
 Full path for system files to perform operations on. Does not apply when used with the purge subcommand (purge removes the entire directory). Defaults to /etc/passwd, /etc/shadow, /etc/group, /etc/gshadow
--archive-directory=/root/.flashback, -D=/root/.flashback
 The directory where archived system and configuration files reside. Defaults to /root/.flashback
--parallel-workers=10, -w=10
 Number of concurrent connections, set to 1 to serialize. Defaults to 10
--sudo-password-prompt=False, -p=False
 Prompt for a password if required for sudo command execution.
